# Design System

## Visual Direction

Memora should look warm, structured, reflective, and implementation-ready. The visual identity is a calm library on parchment, with book, shelf, lantern, chapter, and librarian motifs used to support product meaning.

The UI should avoid feeling like a cold dashboard, a medical app, or a generic SaaS landing page.

## Color Palette

- Parchment background: `#FDF0D6`
- Burgundy primary text and CTA: `#6F241E`
- Terracotta secondary accent: `#B85B4B`
- Dusty rose soft surfaces: `#D7A39E`
- Gold highlights: `#E3B247`
- Teal insight accent: `#4C7C8E`

Burgundy should carry primary actions and strong headings. Parchment should carry the base atmosphere. Gold and teal should be used sparingly for highlights and insights.

## Typography

- Display: Cormorant Garamond
- UI and body: Inter
- H1: 56
- H2: 40
- H3: 28
- Body: 18
- Caption: 14

Display type should be reserved for moments that benefit from the literary identity. Dense controls, inputs, filters, and helper text should use Inter.

## Layout

- Use an 8-point spacing system.
- Use 24 or 32 pixel section gutters.
- Use 16 pixel component/card padding as a baseline.
- Desktop grid: 12 columns.
- Tablet grid: 8 columns.
- Mobile grid: 4 columns.

## Core Components

- Primary button.
- Secondary button.
- Ghost button.
- Entry card.
- Emotion chips: Happy, Stressed, Proud, Disappointed, plus extensible variants such as Grateful, Anxious, Frustrated, and Other.
- Input field.
- Textarea.
- Validation error.
- Focus state.
- Top navigation.
- App sidebar.
- AI librarian card.
- Tone chips: Motivational, Humorous, Wise.

## Interaction Patterns

- Search and filter by emotion, tag, and life phase.
- Shelf browsing with progressive disclosure.
- Conversational reflection through an AI side panel.
- Dashboard insight cards for patterns and revisit prompts.
- Empty states with one clear next action.

## Design Constraints

- Emotional data must feel private and editable.
- AI guidance must be supportive, not therapeutic advice.
- Tone personalization must stay user-controlled.
- The experience should be calm, clear, and low-friction.
