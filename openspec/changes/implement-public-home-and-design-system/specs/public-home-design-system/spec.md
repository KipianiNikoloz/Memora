## ADDED Requirements

### Requirement: Public homepage
The system SHALL provide a responsive public homepage that communicates Memora's value proposition and includes clear sign-up paths.

#### Scenario: Visitor lands on homepage
- **WHEN** a visitor opens the public home route
- **THEN** they see the message "Every life deserves a library" and understand Memora's memory-to-library concept

### Requirement: Shared design system
The system SHALL provide reusable UI components and styling that match the documented Memora design system.

#### Scenario: Component is reused
- **WHEN** product screens need buttons, inputs, chips, cards, panels, navigation, or states
- **THEN** they use shared components with consistent Memora styling

### Requirement: Responsive visual quality
The system SHALL render public and app screens cleanly across mobile and desktop viewports.

#### Scenario: Mobile viewport
- **WHEN** a user views Memora on a mobile viewport
- **THEN** text, controls, and content remain readable and non-overlapping
