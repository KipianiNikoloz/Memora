## Context

The UI deck defines the homepage, design system colors, typography, components, and the warm library atmosphere.

## Goals / Non-Goals

**Goals:**

- Build the homepage around "Every life deserves a library."
- Create reusable buttons, inputs, chips, cards, panels, nav, and state components.
- Apply responsive design to public and app surfaces.
- Preserve the calm, reflective brand identity.

**Non-Goals:**

- No marketing CMS.
- No new auth or data behavior.
- No decorative-only redesign that weakens usability.

## Decisions

- Use the documented palette and typography as hard design constraints.
- Keep homepage first viewport product-specific and trust-building.
- Reuse shared components across app screens.
- Verify mobile and desktop layouts.

## Risks / Trade-offs

- Strong visual style can harm density -> Keep app surfaces scannable and calm.
- Homepage can become too marketing-heavy -> Keep literal product explanation visible.
